Завершение сессий

Завершение активной сессии

select pg_cancel_backend('<Идентиифкатор сессии>');

Жесткое завершение сессии

select pg_terminate_backend('<Идентиифкатор сессии>');

Завершение всех активных подключений в базе

select pg_terminate_backend(pg_stat_activity.pid) from pg_stat_activity where pg_stat_activity.datname = '<имя базы>' and pid <> pg_backend_pid();

pg_cancel_backend может отменить запущенный запрос, а pg_terminate_backend "убить". Когда приложение создает соединение с базой и отправляет запросы, то с помощью pg_cancel_backend можно попытаться отменить выполняющийся запрос без завершений всего соединения. Если полностью завершить сессию (соединение), то все запросы будут остановлены для этого процесса.
