Установка Rocky Linux WSL2

Использование WSL2 для работы с сервером PostgreSQL

Скачиваем дистрибутив RockyLinux на выбор

  • base

  • minimal

Для развертывания общий синтаксис:

wsl --import <machine-name> <path-to-vm-dir> <path-to/rocky-9-image.tar.xz> --version 2

Фактическое расположение с нашем случае

  • где разворачиваем WSL образ d:\wslVM\rocky9\

  • где ранее скачанный образ d:\wslDistroStorage\rocky9\Rocky-9-Container-Base.latest.x86_64.tar.xz

выполняем установку с использованием Powershell

wsl --import rocky9 "d:\wslVM\rocky9\" "d:\wslDistroStorage\rocky9\Rocky-9-Container-Base.latest.x86_64.tar.xz"

в результате видим

wsl --import rocky9 "d:\wslVM\rocky9\" "d:\wslDistroStorage\rocky9\Rocky-9-Container-Base.latest.x86_64.tar.xz" Выполняется импорт. Это может занять несколько минут. Операция успешно завершена.

Установка завершена. Проверяем что установлено:

wsl --list

результат:

wsl --list Дистрибутивы подсистемы Windows для Linux: Ubuntu (по умолчанию) rocky9

Настраиваем терминал: скачиваем иконку, настраиваем сам профиль или вручную или добавим в JSON конфигурации

{ "colorScheme": "Tango Dark", "commandline": "C:\\Windows\\system32\\wsl.exe -d rocky9", "cursorShape": "filledBox", "font": { "face": "Cascadia Mono", "size": 13.0 }, "guid": "{3b9eb47e-c9f0-4e70-ab29-a83f539715f3}", "hidden": false, "icon": "d:\\wslDistroStorage\\rocky9\\fedora-logo-icon.png", "name": "Rocky9", "tabTitle": "Rocky9" }

Обновляем систему

dnf update

Устанавливаем только клиента, по факту мы ставим сервер, но не выполняем конфигурацию сервера, не инициализируем базу и службы

dnf install -y https://download.postgresql.org/pub/repos/yum/reporpms/EL-9-x86_64/pgdg-redhat-repo-latest.noarch.rpm dnf -qy module disable postgresql
dnf install -y postgresql16-server

Далее проверяем:

psql -V
