Сброс пароля

В PostgreSQL суперпользователем является postgres. Если вы забыли пароль postgres, вы можете сбросить его, выполнив следующие действия.

Найдите файл конфигурации pg_hba.conf для сервера базы данных PostgreSQL.
В Windows файлы конфигурации сервера базы данных PostgreSQL находятся в каталоге данных каталога установки PostgreSQL, например:
C:\Program Files\PostgreSQL\14\data

В Linux файл конфигурации сервера базы данных PostgreSQL находится по адресу
/etc/postgresql/14/main/pg_hba.conf

Сделайте резервное копирование файла конфигурации pg_hba.conf перед его изменением, чтобы вы могли восстановить его позже.

cp pg_hba.conf pg_hba.conf.bak

Изменение файла конфигурации для доверия к локальным соединениям не требующие пароля. Изменить scram-sha-256 или md5 в файле конфигурации для доверия следующим образом:

local all all peer # IPv4 local connections: host all all 127.0.0.1/32 trust # IPv6 local connections: host all all ::1/128 trust # Allow replication connections from localhost, by a user with the # replication privilege. local replication all peer host replication all 127.0.0.1/32 trust host replication all ::1/128 trust

  • Перезапустите сервер базы данных PostgreSQL.

В Windows перезапустить PostgreSQL можно в окне Списка служб.
В Linux вы можете перезапустить PostgreSQL с помощью команды systemctl restart postgresql.

  • Войдите на сервер базы данных PostgreSQL.

psql -U postgres

Вам не нужно будет вводит пароль!!!

Для изменения пароля пользователя postgres используйте следующую команду:

ALTER USER postgres WITH PASSWORD 'new_password';

Восстановите файл конфигурации pg_hba.conf, перезаписать файл pg_hba.conf содержимым файла pg_hba.conf.bak

Перезапустите сервер базы данных PostgreSQL. При входе в систему PostgreSQL должен запросить пароль.
