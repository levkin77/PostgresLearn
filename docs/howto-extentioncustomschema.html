Установка extention в отдельную схему

Отдельная схема для всех расширений

После установки extention в базе создаются дополнительные объекты. Обычно это приводит к захломлению в схеме public.

Как вариант можно устанавливать все extention в отдельную схему.

create schema extensions; -- make sure everybody can use everything in the extensions schema grant usage on schema extensions to public; grant execute on all functions in schema extensions to public; -- include future extensions alter default privileges in schema extensions grant execute on functions to public; alter default privileges in schema extensions grant usage on types to public;

Теперь для установки Extention используем

create extension unaccent schema extensions;

Для включения в search path

set search_path = public, extensions;

Для проверки search path

show search_path;

Например, установим extension tablefunc

create extension tablefunc schema extensions;

и проверим, что за новые функции появились в нашей схеме extensions

select v.routine_schema, v.routine_name, v.external_language, v.routine_type, case when count(*)>1 then count(*) else 0 end as hasreload from information_schema.routines v where v.routine_schema='extensions' group by v.routine_schema, v.routine_name, v.external_language, v.routine_type;

Если вы не хотите повторять вышесказанное для каждой новой базы данных, которую вы создаете, выполните вышеуказанные шаги при подключении к базе данных template1.

Вы даже можете включить схему расширений в путь поиска по умолчанию, отредактировав postgresql.conf или используя alter system.

Или обращаться к необходимым функция по полному имени, например extensions.connectby(...)
