Postgres Полезные русурсы Help

Установка на Ubuntu 22.04

Выполнить обновление системы

sudo apt update && sudo apt upgrade

Установка PostgreSQL Server

sudo apt install postgresql

Если необходима установка конкретной версии

sudo apt install postgresql-16
sudo apt install postgresql-15
sudo apt install postgresql-14

При наличии проблем с установкой на WLS2 прочитать официальную документацию или как вариант попробовать

sudo sh -c 'echo "deb http://apt.postgresql.org/pub/repos/apt $(lsb_release -cs)-pgdg main" > /etc/apt/sources.list.d/pgdg.list' wget --quiet -O - https://www.postgresql.org/media/keys/ACCC4CF8.asc | sudo apt-key add - sudo apt-get update sudo apt-get -y install postgresql

Проверка статуса службы и порт

systemctl status postgresql
ss -nlt

Перезапуск службы

sudo systemctl restart postgresql

Остановка службы

sudo systemctl stop postgresql

Отключение автостарта

sudo systemctl disable postgresql

Включение автостарта

sudo systemctl enable postgresql

Установка только клиента PostgreSQL

sudo apt install postgresql-client

Установка клиента для v16

sudo apt install postgresql-client-16

Установка клиента для v15

sudo apt install postgresql-client-15

Для проверки подключения к серверу

psql -h localhost -d DB-name -U DB-User
psql -h server-ip-address -d DB-name -U DB-User

Безопасность база данных по умолчанию PostgreSQL

Изменение пароля для пользователя nix

sudo passwd postgres

Вход под пользователем nix

su - postgres

Изменение пароля в базе данных для пользователя базы

psql -c "ALTER USER postgres WITH PASSWORD 'your-password';"

Настройка PostgreSQL для удаленного доступа

Изменение прослушиваемого адреса

sudo nano /etc/postgresql/16/main/postgresql.conf

и проверяем наличие строки

listen_addresses = '*'

Для WSL2 можно дополнительно установить редактор текстовых файлов

sudo apt install gedit -y sudo gedit /etc/postgresql/16/main/postgresql.conf

Редактирование pg_hba.conf

sudo nano /etc/postgresql/14/main/pg_hba.conf

и проверяем

host all all 0.0.0.0/0 md5 host all all ::0/0 md5

После изменения настроек

sudo systemctl restart postgresql

Проверьте, слушает ли postgreSql все адреса или нет, для этого вы можете запустить:

sudo apt install net-tools sudo netstat -antup | grep 5432

Результат

tcp 0 0 0.0.0.0:5432 0.0.0.0:* LISTEN 11568/postgres tcp6 0 0 :::5432 :::* LISTEN 11568/postgres

Разрешаем порт

sudo ufw allow 5432

Удаление PostgreSQL

sudo apt remove postgresql*

Для удаления версии

sudo apt remove postgresql-16*

Для удаления только клиента

sudo apt remove postgresql-client
