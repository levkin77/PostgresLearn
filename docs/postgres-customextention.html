Ресурсы и ссылки на дополнительные расширения

Работаем с Сlickhouse clickhouse_fdw

plpgsql_check

  • проверяет поля ссылочных объектов и типов баз данных внутри встроенного SQL

  • подтверждает, что вы используете правильные типы для параметров функции

  • идентифицирует неиспользуемые переменные и аргументы функции, неизмененные аргументы OUT

  • частичное обнаружение мертвого кода (код после команды RETURN)

  • обнаружение отсутствующей команды RETURN в функции (общие после обработчиков исключений, сложная логика)

  • пытается идентифицировать нежелательные скрытые слепки, которые могут быть проблемой производительности, как неиспользованные индексы

  • способность собирать отношения и функции, используемые функцией

  • возможность проверять операторы EXECUTE на предмет уязвимости SQL-инъекций

pg_cron

pg_cron - это простой планировщик заданий на основе cron для PostgreSQL, который работает внутри базы данных в качестве расширения. Он использует тот же синтаксис, что и обычный cron, но позволяет планировать команды PostgreSQL непосредственно из базы данных.

pgagent

pgagent - планировщик работы для PostgreSQL. pgAgent управляется с помощью pgAdmin (http://www.pgadmin.org). PgAdmin документация содержит подробную информацию о настройке и использовании pgAgent с PostgreSQL. Последнюю сборку документации можно найти по адресу https://www.pgadmin.org/docs/pgadmin4/development/pgagent.html.

pg_profile

Утилита стратегического мониторинга. Это расширение поможет вам найти наиболее ресурсоёмкие действия в ваших базах данных. Дополнительное видео о расширении pg_profile - видео
