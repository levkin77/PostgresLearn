Применение столбцов updated_by и created_by

Одним из способов анализа "кто обновил запись" или "кто создал" в таблице является использование столбца updated_by или created_by. Это достаточно удобный вариант, быстро реализуемый и имеет очевидные преимущества.

Данные updated_by - в текстовом виде

Концепция хранения данных в виде простого текстового поля, в качестве значений которого выступают фактические наименования реальных пользователей базы данных.

Естественным недостатком является наличие потенциального списка пользователей в базе данных непосредственно в таблице.

Связь между таблицами - только логическая.

Данные updated_by - в виде сссылки на ключ пользователей

Концепция хранения данных в виде ссылки на данные о таблице пользователей, в качестве значений которого выступают первичные ключи самой таблицы пользователей. Может быть реализована как при наличии отдельной таблицы пользователей приложения, так и использую внутренни идентификаторы пользователей базы данных.

При реализации через связь на системную таблицу баз данных пользователей можно получить проблемы с переносимостью базы на другие сервера.

Связь между таблицами - может быть как логическая так и физическая, но лучше только логическая.

Данные updated_by - в виде ссылки на код пользователей

Концепция хранения данных в виде ссылки на данные о таблице пользователей, в качестве значений которого выступают не первичные ключи таблицы пользователей, а столбец принадлежащий таблице пользователей, но являющийся албтернативой первичному ключу (например полученный с помощью использования хеширования фактического имени пользователя). Связь между таблицами - только логическая.
